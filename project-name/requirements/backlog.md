# Requirements Backlog

아직 정제되지 않은 요구, 질문, 아이디어를 여기에 빠르게 적재한다.

## intake template

| Date | Candidate | Problem | Source | Next Step |
| --- | --- | --- | --- | --- |
| YYYY-MM-DD | 예: 관리자 승인 플로우 | 승인 기준이 문서마다 다름 | user request / wiki/foo.md | 요구사항 문서로 정제 |

## rules

- backlog는 자유롭게 추가하되, 구현 지시로 바로 사용하지 않는다.
- 중복 후보는 병합하고, 병합 사실은 관련 커밋 또는 요구사항 문서에서 명확히 남긴다.
- 구현 착수 전에는 반드시 정제된 요구사항 문서로 승격한다.
