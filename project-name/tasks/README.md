# Tasks

이 영역은 요구사항을 실제 구현 가능한 작업 단위로 분해해 관리한다.

## purpose

- 요구사항을 구현 순서와 책임 단위로 쪼갠다.
- 진행 상태를 요구사항과 별도로 관리한다.
- 코드 변경, 테스트, 문서 작업을 명시적으로 연결한다.

## rules

- 각 task는 하나 이상의 요구사항 ID를 참조한다.
- task는 요구사항을 복제하지 않고, 구현 단위와 완료 조건만 적는다.
- 구현 착수 전에는 관련 요구사항 상태가 최소 `approved`여야 한다.
- 완료된 task는 관련 코드 또는 산출물을 남겨야 한다.

## status

- `todo`
- `in_progress`
- `blocked`
- `done`
